# DATASET.md

## 1. Purpose

CDF creates synthetic, constant-thickness sheet-metal samples for AMG training. The dataset teaches AMG to predict an AMG-compatible mesh-control manifest, not mesh node coordinates or element connectivity.

## 2. Accepted sample definition

A sample is accepted only when all checks pass.

```text
valid CAD solid
constant thickness validation
feature placement constraints
truth-to-detected feature matching
AMG_MANIFEST_SM_V1 schema validation
ANSA oracle pass when oracle is enabled
```

The oracle mesh is stored as evidence. It is not the supervised target.

## 3. Dataset root structure

```text
dataset_root/
  dataset_index.json
  dataset_stats.json
  config_used.json
  contracts/
  splits/
  rejected/
  samples/
```

Every accepted sample contains:

```text
cad/input.step
cad/reference_midsurface.step
metadata/generator_params.json
metadata/feature_truth.json
metadata/entity_signatures.json
graph/brep_graph.npz
graph/graph_schema.json
labels/amg_manifest.json
labels/face_labels.json
labels/edge_labels.json
labels/feature_labels.json
meshes/ansa_oracle_mesh.bdf
reports/ansa_execution_report.json
reports/ansa_quality_report.json
reports/sample_acceptance.json
```

## 4. Feature truth

Feature truth is generated at CAD operation time. It is not manually labeled.

Required fields:

```text
feature_id
type
role
geometry parameters
patch_id or adjacent_patch_ids
created_by
```

Example:

```json
{
  "feature_id": "HOLE_BOLT_0001",
  "type": "HOLE",
  "role": "BOLT",
  "center_mm": [42.0, 30.0, 0.0],
  "axis": [0.0, 0.0, 1.0],
  "radius_mm": 3.2,
  "created_by": "cadgen.circular_cut"
}
```

## 5. Label generation

CDF labels are deterministic.

```text
CAD parameters + feature truth + config
  → AMG-compatible rule label
  → labels/amg_manifest.json
```

ANSA oracle does not modify labels. Failed oracle samples are rejected or stored under rejected records.

## 6. Label leakage rule

Graph input files must not contain target labels.

Allowed in `feature_features.npy`:

```text
feature_type_id
role_id
geometry-derived sizes
clearance metrics
expected_action_mask
```

Not allowed in graph inputs:

```text
target_action_id
target_edge_length_mm
circumferential_divisions
washer_rings
bend_rows
labels copied from labels/*.json
```

## 7. Splits

Target split files:

```text
splits/train.txt
splits/validation.txt
splits/test_seen.txt
splits/test_l_bracket_holdout.txt
splits/test_u_channel_holdout.txt
splits/test_dense_perforation_holdout.txt
```

Rules:

```text
- Splits contain sample IDs, one per line.
- A sample appears in exactly one split.
- Holdout splits are generated by parameter/class filters, not random leakage.
- config_used.json stores split seed and split policy.
```

## 8. Dataset stats

`dataset_stats.json` should include:

```text
num_accepted
num_rejected
part_class_counts
feature_type_counts
feature_role_counts
action_counts
h0_distribution
thickness_distribution
ANSA oracle pass/fail counts
rejection_reason_counts
```

## 9. Rejected samples

Rejected samples are useful for debugging but not used as accepted training data unless a future task explicitly defines negative training.

`rejected/rejected_index.json` records:

```text
sample_attempt_id
reason
phase
partial_artifacts
metrics
message
```

## 10. Minimum smoke dataset

A valid smoke dataset for early development can contain 10 accepted flat-panel samples with one or more holes. It must still satisfy schemas and label leakage rules.
